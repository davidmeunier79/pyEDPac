To run this on a 64-core cluster node, you will treat each Individual as a standalone object that encapsulates both its physical state (position in the Zoo) and its mental state (its EvoNetwork).

class Agent:
    def __init__(self, x, y, zoo, genome, species="prey"):
        self.animal = Animal(x, y, zoo)
        self.brain = EvoNetwork(genome)
        self.species = species
        self.alive = True
        self.fitness = 0

    def update(self):
        if not self.alive: return
        # 1. Sense: Get local vision from the zoo
        obs = self.animal.get_vision()
        # 2. Think: Pass to EvoNetwork
        action = self.brain.compute(obs)
        # 3. Act: Move animal based on output
        self.animal.move(action)

2. Parallelization Strategy: "Island" vs. "Single-World"

There are two ways to interpret "using 64 cores":

    Option A: 64 Independent Worlds (Parallel) You run 64 different Zoo instances. Each core handles 1 Prey vs 1 Predator in their own private map. This is what joblib does best.

    Option B: 1 Massive World (Synchronous)
    All 64 agents are in one large Zoo. You cannot easily use joblib inside a single world step because agents interact (Prey A gets eaten by Predator B).

For a Cluster/Joblib setup, Option A is the way to go. You evaluate "duels" or "small groups" in parallel.

This is a sophisticated "System-on-Chip" approach to neural simulation. By using a single Global Event Manager to orchestrate 64 different EvoNetwork instances, you effectively treat the entire Zoo as one massive, heterogeneous asynchronous circuit.

To make Option B work with a priority queue, you need to "Tag" every event with an Agent ID. This allows the Event Manager to route Post-Synaptic Potentials (PSPs) to the correct neuron in the correct brain.
1. The Global Event Manager Structure

Your PSPEvent needs to be extended. Instead of just (time, neuron_id, weight), it should follow a hierarchical addressing scheme: Agent ID → Population ID → Neuron ID.
Python

# Modified Event Structure
class GlobalPSPEvent:
    def __init__(self, time, agent_id, pop_id, neuron_id, weight):
        self.time = time
        self.agent_id = agent_id  # Which prey/predator
        self.pop_id = pop_id      # e.g., 0 for Sensory, 1 for Interneuron, 2 for Motor
        self.neuron_id = neuron_id
        self.weight = weight

    def __lt__(self, other):
        return self.time < other.time

2. Orchestration Logic: The "Tickless" Loop

In this setup, the "Zoo" doesn't just loop through agents; it acts as an I/O Bridge. It injects "Sensory Events" into the queue and listens for "Motor Events" coming out.

The Simulation Heartbeat:

    Sense: The Zoo scans the environment for all 64 agents. It generates PSPEvents for the Sensory Neurons of each agent and pushes them into the global Priority Queue.

    Process: The Event Manager runs until a specific SYNC_TIME (the motor integration window).

    Act: Any spikes that occurred in the "Motor Population" of an agent are collected. The Zoo translates these spikes into physical displacement (Up, Down, Left, Right).

    Resolve: The Zoo checks for collisions (Predator eats Prey). Dead agents are flagged, and their events are purged from the queue.

3. Handling Multiple Brains in one Queue

To implement this, you should modify your EvoNetwork to be "stateless" regarding time, or pass the Global Event Manager into the network's spike method.
Python

def process_global_queue(event_manager, agents, sync_step=1.0):
    """
    Processes all neural activity for all 64 agents
    up to the next physical world tick.
    """
    target_time = event_manager.get_time() + sync_step

    while not event_manager.empty() and event_manager.peek_time() < target_time:
        event = event_manager.pop()

        # Route the event to the specific brain
        agent = agents[event.agent_id]
        if agent.alive:
            # The brain processes the PSP and potentially
            # schedules NEW events into the global event_manager
            agent.brain.receive_psp(event)

5. Bottleneck Warning: The "Serial" Queue

The main challenge with Option B is that a Priority Queue is inherently serial. Even on 64 cores, the Event Manager usually runs on one core because the order of events must be strictly maintained.

How to exploit your 64 cores in Option B:

    Parallel Sensory Processing: Use joblib/multiprocessing to calculate the "Vision" (ray-casting/grid scanning) for all 64 agents in parallel. This is usually the most expensive CPU task.

    Parallel Brain Update: If your neural model allows for small synchronization delays, you can partition the Event Manager into 64 "Sub-Queues" (one per agent) and run them on 64 cores, synchronizing only at the sync_step.

When two individuals of the same species occupy adjacent cells (or the same cell), a "Mating Event" is triggered. This requires your Zoo and Agent logic to handle real-time genome crossover and "Spawning."
1. The Mating Trigger Logic

Within your live_one_step or check_collisions method, you need a secondary check for intra-species contact.

def handle_interactions(self):
    # 1. Predator vs Prey (The Hunt)
    for pred in self.predators:
        for prey in self.preys:
            if self.is_touching(pred, prey):
                prey.alive = False
                pred.fitness += 50
                self.remove_agent(prey)

    # 2. Intra-species Contact (The Mating)
    for pop in [self.preys, self.predators]:
        for i, agent_a in enumerate(pop):
            for agent_b in pop[i+1:]:
                if self.is_touching(agent_a, agent_b):
                    if agent_a.can_mate() and agent_b.can_mate():
                        self.reproduce(agent_a, agent_b, pop)

When two agents mate, you perform the crossover of their EvoNetwork genomes immediately and place a new Agent into the world.

def reproduce(self, parent_a, parent_b, population_list):
    # 1. Crossover Genomes
    child_genome = crossover(parent_a.genome, parent_b.genome)
    child_genome = mutate(child_genome) # Keep diversity high

    # 2. Create New Agent
    # Place it near the parents
    spawn_x, spawn_y = self.find_empty_neighbor(parent_a.x, parent_a.y)
    child = Agent(spawn_x, spawn_y, self.zoo, child_genome, species=parent_a.species)

    # 3. Add to the Global Simulation
    population_list.append(child)

    # 4. Set "Cool-down" to prevent infinite reproduction loops
    parent_a.reset_mating_timer()
    parent_b.reset_mating_timer()

3. Managing the Population Cap (The "Node" Limit)

Since you mentioned the simulation is driven by the number of cores (e.g., 64), you must prevent Population Explosion. If 64 agents keep mating, you'll eventually have 5,000 agents, and your event manager will crawl to a halt.

Strategies for Steady-State Balance:

    Death by Age: Give every agent a max_lifespan. This ensures "Old" genomes eventually clear space for "New" ones.

    Hard Cap: If len(all_agents) >= 64, mating events simply fail.

To implement a high-performance multi-agent system where the **Zoo** is the master and **EvoNetworks** are distributed workers, we use `multiprocessing.Pipe`. Pipes provide a fast, duplex (two-way) communication channel that is more efficient than a shared Queue for 1-to-1 synchronization.

In this architecture, the Zoo stays in the main process (handling the grid, collisions, and GA), while each brain lives in its own dedicated process.



### 1. The Worker Process (The Brain)
Each worker waits for sensory data, runs the neural inference, and sends back motor commands. It only receives a new chromosome at the start of a generation.

```python
import multiprocessing as mp
import numpy as np
from edpac.network.evo_network import EvoNetwork

def brain_worker(pipe, agent_id):
    """
    Independent process for one agent's brain.
    """
    network = None

    while True:
        # Wait for a message from the Zoo
        msg = pipe.recv()

        cmd = msg.get('type')

        if cmd == 'SET_CHROMOSOME':
            # Initialize/Update the network with a new genome
            network = EvoNetwork(msg['data'])
            pipe.send({'type': 'READY'})

        elif cmd == 'SENSE':
            # Perform neural computation
            vision_input = msg['data']
            motor_output = network.compute(vision_input)
            # Send motor command back to Zoo
            pipe.send({'type': 'MOTOR', 'data': motor_output})

        elif cmd == 'TERMINATE':
            break
```

### 2. The Master Process (The Zoo)
The Zoo maintains a list of `Process` objects and a list of `Connection` objects (the "parent" end of the pipes).



```python
class CentralizedZoo:
    def __init__(self, population_size=64):
        self.size = population_size
        self.pipes = []
        self.workers = []
        self.agents = [] # Physical representations in the Zoo

    def setup_workers(self):
        for i in range(self.size):
            parent_conn, child_conn = mp.Pipe()
            p = mp.Process(target=brain_worker, args=(child_conn, i))
            p.start()
            self.pipes.append(parent_conn)
            self.workers.append(p)

    def run_generation(self, genomes):
        # 1. Send chromosomes to all workers
        for i, pipe in enumerate(self.pipes):
            pipe.send({'type': 'SET_CHROMOSOME', 'data': genomes[i]})

        # Wait for all to be ready
        for pipe in self.pipes:
            pipe.recv()

        # 2. Main Simulation Loop
        for tick in range(1000):
            # A. Distribute Vision (Master -> Workers)
            for i, pipe in enumerate(self.pipes):
                vision = self.get_agent_vision(i) # From Zoo grid
                pipe.send({'type': 'SENSE', 'data': vision})

            # B. Collect Motor Commands (Workers -> Master)
            # This is the synchronization point
            for i, pipe in enumerate(self.pipes):
                response = pipe.recv()
                motor = response['data']
                self.apply_action(i, motor)

            # C. Update World Physics & Collisions
            self.resolve_physics()

            if self.all_dead():
                break

        # 3. Collect Fitness & Run GA
        scores = [a.fitness for a in self.agents]
        new_genomes = self.genetic_algorithm(genomes, scores)
        return new_genomes

    def cleanup(self):
        for pipe in self.pipes:
            pipe.send({'type': 'TERMINATE'})
        for w in self.workers:
            w.join()
```

### 3. Critical Considerations for Cluster Performance

#### A. The Synchronization Bottleneck
In this model, the Zoo must wait for the **slowest** brain to finish its `compute()` before the physics tick can resolve. If one `EvoNetwork` has 1,000 neurons and another has 10, the 1,000-neuron one will dictate the frame rate.
* **Fix:** Ensure your genomes have a complexity penalty or a hard cap on neuron count.

#### B. Data Serialization (Pickling)
Every time you `pipe.send()`, Python "pickles" the object.
* **Optimization:** Do not send complex Python objects. Send **NumPy arrays** or **integers**. Sending a large vision grid as a raw list of strings will be 10x slower than sending it as a `np.uint8` array.

#### C. Mating and Mid-Generation Spawning
Since you want reproduction to happen on contact:
1.  When Agent A and B touch, the **Zoo** performs the crossover.
2.  The Zoo selects an idle pipe (from a "Dead" agent) or spawns a new process.
3.  The Zoo sends the new `SET_CHROMOSOME` command to that pipe **mid-simulation**.
4.  The new brain is now active in the next tick.

This is more complex but allows for **Sub-Tick Parallelism**, where neural spikes happen asynchronously across your 64 cores.
